## 🎨 デザインの特徴

### カラースキーム
- **メインカラー**: ブルー系（#0ea5e9）
- **背景**: グラデーション（ブルーからインディゴ）
- **完了状態**: グリーン（#10b981）
- **未完了**: オレンジ（#f97316）

### UIコンポーネント

#### 1. 統計カード（4枚）
- 総数（ブルー）
- 未完了（オレンジ）
- 完了済み（グリーン）
- 達成率（パープル）

#### 2. Todo作成フォーム
- タイトル入力（必須）
- 説明入力（オプション、トグル表示）
- カテゴリ入力（オプション、トグル表示）
- 追加ボタン（ブルー）

#### 3. フィルターボタン
- すべて / 未完了 / 完了済み
- アクティブなボタンはブルーで強調

#### 4. Todoアイテムカード
- チェックボックス（完了切替）
- タイトル（完了時は打ち消し線）
- 説明（グレーテキスト）
- カテゴリタグ（ブルーバッジ）
- 作成日時
- 削除ボタン（レッドアイコン）

## 🖱️ インタラクション

### ユーザーができること

1. **Todoを追加**
   - タイトルを入力して「追加」ボタンをクリック
   - 「詳細を表示」で説明やカテゴリも入力可能

2. **Todoを完了にする**
   - チェックボックスをクリック
   - タイトルに打ち消し線が表示される

3. **Todoを削除**
   - 🗑️ アイコンをクリック
   - 即座にリストから削除

4. **フィルタリング**
   - 「すべて」「未完了」「完了済み」ボタンで切り替え
   - リアルタイムで表示が更新

5. **統計を確認**
   - ページ上部のカードで全体の進捗を把握
   - Todoの追加・完了でリアルタイム更新

## 📐 レスポンシブデザイン

### デスクトップ（1024px以上）
- 統計カードは4列で表示
- Todoカードは1列、横幅最大4xl（896px）

### タブレット（768px〜1023px）
- 統計カードは2列で表示
- その他はデスクトップと同様

### スマートフォン（767px以下）
- 統計カードは1列で縦に並ぶ
- Todoカードも1列で表示
- ボタンやフォームは画面幅に合わせて調整

## 🔍 確認方法

### 1. サーバーを起動して確認（推奨）

```bash
# Docker Composeで起動
docker-compose up --build

# ブラウザで開く
http://localhost:3000
```

### 2. 開発者ツールで確認

- **Chrome/Edge**: F12 キー
- **レスポンシブモード**: Ctrl+Shift+M (Windows) / Cmd+Shift+M (Mac)
- **コンソール**: エラーやログを確認
- **ネットワーク**: APIリクエストを確認

### 3. 各ブレークポイントで確認

開発者ツールのレスポンシブモードで以下のサイズをテスト:
- モバイル: 375px × 667px (iPhone SE)
- タブレット: 768px × 1024px (iPad)
- デスクトップ: 1920px × 1080px (Full HD)

## 💡 カスタマイズ方法

### カラーを変更

`frontend/tailwind.config.js` で色を変更:

```javascript
theme: {
  extend: {
    colors: {
      primary: {
        500: '#0ea5e9', // ここを変更
      },
    },
  },
}
```

### フォントを変更

`frontend/src/app/layout.tsx` でフォントを変更:

```typescript
import { Inter } from 'next/font/google'
// 他のGoogle Fontsに変更可能
```
